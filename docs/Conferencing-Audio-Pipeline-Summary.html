Conferencing Audio Pipeline — Work Summary

A vendor-neutral, Shure-Designer-style offline audio-configuration control plane: a pure-Python engine (conf_pipeline) plus a PySide6 2D/3D layout editor (conf_pipeline_gui). It models a networked conferencing audio system — mic coverage zones → matrix mixer → AEC references + automixer → outputs — and validates correctness, above all the AEC self-reference rule. It does not process, mix, cancel, or stream real audio.

Architectural principles (held throughout)

Immutable builders. Every engine function returns a new SystemConfig; nothing is mutated in place. The GUI commits a change via state.set_config(new), which is exactly one undo step.
TS-compatible JSON round-trip. to_jsonable / config_from_dict convert to camelCase; optional None fields are auto-omitted unless listed in _NULLABLE_KEYS.
AEC self-reference rule (cardinal invariant). A mic's AEC reference bus must never contain that mic's own signal. Enforced by analyze_aec_reference / contains_own_signal; error codes AEC_SELF_REFERENCE and AEC_REINFORCED_SHARED_REFERENCE.
Processor bus convention. A processor output bus has bus.id == bus.port_id, so a bus id works both as a matrix output column and as a route from-port.
Pure stdlib engine. No numpy/Qt in the engine. numpy / pyroomacoustics are imported only inside the optional physics-validation backends, behind availability gates.

Tasks & Features

▶TASK 1Placement simulation & recommendationv1.9.0
"Add simulations that tell me the best possible angle to place my mic setup and where the person should sit."

Decisions: hybrid fidelity (fast heuristic sweep + optional physics validation of the top pick), joint optimization (array placement and seat), all four objectives, GUI panel + engine API, pluggable validation backend.

New package conf_pipeline/sim/:

  • scoring.py — four objectives blended into one score: direct-path level/SNR (inverse-distance spreading + main-lobe rolloff); direct-to-reverberant ratio (Sabine RT60 → critical distance); coverage / on-axis (gaussian lobe gated by pickup/exclusion zones); multi-talker fairness (mean / worst / variance). Plus estimated_rt60, score_placement.
  • search.py — recommend_placement (joint array-pose + seat, coarse-to-fine, steer derived analytically, min-separation from other talkers) and score_heatmap (where-to-mount-the-array grid).
  • validate.py — pluggable physics backends: farfield (numpy plane-wave UCA delay-and-sum) and pyroomacoustics (image-source RIR: physical DRR + beam SNR). Plus available_backends, numpy_available, validate_recommendation.

F3 — Coverage areas + checks

  • conf_pipeline/profiles.py — added coverage_angle_deg: Optional[float] = None to DeviceCapabilities (ceiling array 120°, table array 130°; not serialized — catalog is static).
  • conf_pipeline/coverage_check.py (new) — geometric coverage:
    • array_coverage_radius(mount, target, angle) = max(mount−target, 0)·tan(angle/2)
    • array_coverage_circle(config, array_id) -> (center, radius)
    • coverage_report(config) -> CoverageReport(covered, uncovered, overlaps)
    • Reuses default_elevation, point_in_shape, device_capabilities; mirrors the exclusion-zone logic of talker_coverage.
  • GUI — AppState.show_coverage; a checkable Show coverage toolbar action; _paint_coverage draws dashed circles in each array's style colour; Issues tab gains a read-only Coverage group. Advisory only — not new validate() codes.

F2 — Auto-Route (one-click optimize) + summary

  • conf_pipeline/api.py — AutoRouteResult dataclass, _pick_program_bus, auto_route(config). Layered on auto_configure: feeds far-end → loudspeakers, matrix-routes far_end → program bus, routes bus.port_id → speaker input, creates a deterministic synced mic mute-link (ml:auto:{bus}), returns a human-readable change list. Returns "No changes." when re-run on its own output.
  • AEC-safe — a forbidden output-column set keeps the AEC self-reference rule intact; locked by a zero-errors test.
  • Idempotency fix to auto_configure — it now reuses any existing reference/automix buses instead of picking fresh ones on re-run:
existing_ref = next((m.aec.reference_bus_id for m in mics
                     if m.aec.enabled and m.aec.reference_bus_id), None)
existing_automix = config.automixer.output_bus_id
pick_iter = iter(_pick_unused_dante_output_buses(new, processor, 2))
ref_bus = existing_ref or next(pick_iter, None)
automix_bus = existing_automix or next((b for b in pick_iter if b != ref_bus), None)
  • GUI — an Auto-Route toolbar button → _auto_route commits one undo step and shows a QMessageBox summary of what changed.

F1 — Floor-plan import + scale calibration

  • conf_pipeline/model.py — RoomBackground dataclass (path, image_width_px, image_height_px, scale_m_per_px=None, origin=Point2D(0,0), opacity=0.5); RoomLayout.background; _bg(d) deserializer. Stored by path, not base64 — to avoid bloating every JSON, undo snapshot, and diff. Pixel dims persisted so the world rect survives a missing file.
  • conf_pipeline/api.py — builders set_room_background, set_room_background_scale, set_room_background_opacity, clear_room_background, plus the pure, unit-tested calibrated_scale(old, world_dist, real_len) = old · real_len / world_dist.
  • GUI — canvas.py caches a QPixmap by path; _paint_background draws it under the grid in 2D (opacity is sticky and must be reset). Floor plan… imports an image; Calibrate… is a drag-a-known-distance gesture, zoom-independent via s2w. Missing image files degrade gracefully.

F4 — Design report export (Markdown + HTML)

  • conf_pipeline/report.py (new) — design_report(config, fmt="markdown"|"html"). Markdown is the source of truth; a tiny in-module converter emits self-contained HTML using only stdlib html.escape (no new dependency). Sections: room + lazy estimated_rt60, device/channel table, routing, AEC references, coverage, validation. Every interpolated label HTML-escaped.
  • GUI — an Export report toolbar action writes .md / .html.

Errors found & fixed

▶FIXESSix issues resolved during development
  • Auto-route not idempotent — re-running produced changes because _pick_unused_dante_output_buses saw previously-picked buses as "used". Fixed by reusing existing reference/automix buses. First-run behaviour unchanged.
  • GUI smoke hung / no output — _auto_route's modal QMessageBox.information blocks in headless mode. Correct for interactive use; the smoke test patches it out.
  • Multi-array fairness test gave equal scores with a free seat → switched the test to array-only mode (talkers fixed).
  • refine_step_m=0 hang → capped max_n=48 in _local_grid.
  • Seat placed on top of a talker → added min_talker_separation_m.
  • DRR pinned to 0 → recalibrated drr_window_db.
  • Six adversarial multi-agent-review findings fixed.
